- 母法：**工會法第 30 條第 2 項**授權訂定（準則第 1 條）。會員查核權見**工會法第 29 條**（準則第 28 條第 2 項引用）；舉辦事業見工會法第 5 條第 9 款（準則第 25 條）。

- ⚠ 真實財務數字**只存在瀏覽器本機（localStorage）＋匯出檔**，絕不寫進公開 GitHub repo（GitHub Pages 公開），以免外洩＝授人於柄。公開「透明版」由幹部自行匯出後決定要不要貼。

| **14** | 財產＝投資＋固定資產＋其他資產。投資種類、比例須會員（代表）大會議決。 |
| **15** | 應訂**財務處理辦法**經大會議決後實施（含財產登記、增置、增減值、處分、負擔、保管運用程序）。 |
| **16** | **不動產**購置/出售/轉讓/負擔/權利設定，須**會員（代表）大會議決**。 |
| **17** | 應訂普通會計、出納會計、財物會計處理程序；從事投資/事業者另訂成本會計等。 |
| **18** | 普通會計程序含：會計事務範圍/執行、憑證處理（列明製作審核人責任）、簿籍處理（列明記載複核人責任）、報告、預決算編製、檔案保管。 |
| **19** | 財物會計＝取得/保管/處分帳務。**零用金不得超過 10 萬**（額度與運用規則經理事會決議，交財務人員保管）。**日常開支每筆 ≤ 1 萬得在零用金以現金支付**。**財物須以工會名義登記，不得登記他人名下，不得挪為私用**。 |
| **20** | 收入應有正式收據存根或可資證明書表憑證。**提用存款時，理事長、秘書長及會計出納人員須於領款憑證上共同蓋章**並承擔用印法律責任；未設秘書長者由其他相關職務人員蓋章。 |
| **21** | **年度事業費＋辦公費支出不得少於總支出 40%**，覈實用人。會務人員待遇表由理事會配合年度預算訂定並經大會議決。 |
| **22** | 應**逐年提列足夠準備金**，金額由理事會訂定經大會議決；但**決算虧損者得不提列**。 |

## 2. 附表 — 會計科目及說明表（第7條依據）

5 大類官方科目（程式登帳下拉選單＝此清單，不可自創違反附表）：

- **資產類**：流動資產（零用金、銀行存款、短期投資、應收款、用品盤存、暫付款、預付費用、進項稅額、應退稅款、代付款、其他流動資產、基金及投資〔基金/長期投資〕、備抵壞帳）；固定資產（土地、房屋及建築、運輸設備、辦公設備、其他設備、累積折舊）；其他資產（存出保證金、遞延費用、其他）。
- **負債類**：流動負債（應付票據、應付款、應付費用、應付稅款、其他應付款、短期借款、長期借款、暫收款、銷項稅額、代收款項、其他流動負債）；其他負債（存入保證金、預收款項、其他）。

- **支出類**：人事費、辦公費、事業費、雜項購置、特別費、退休撫恤費、預備金、其他支出。

## 3. 工具設計（finance.html，與站內風格一致：紅/金/綠、Noto TC、卡片式、max 600px、PWA）

**兩種臉、同一工具（沿用 evidence.html 模式）：**

### A. 幹部後台（記帳）
- 逐筆收支登記：日期、摘要、**會計科目（附表下拉，標大類）**、收/支、金額、憑證號、經辦/審核人、是否現金（零用金）、是否政府補助。
- 即時**法遵檢核（每項標條號）**：
  - 第19條：零用金餘額 ≤ 10 萬；單筆現金支出 ≤ 1 萬才可走零用金。
  - 第21條：事業費＋辦公費 ÷ 總支出 ≥ 40%（未達標紅字警示）。
  - 第5條：金額為新臺幣整數元。
  - 第20條：支出（提存款）提示需理事長＋秘書長＋會計出納共同蓋章。
  - 第23條：結餘不得分配（鎖死「盈餘分配」類動作、給警語）。
  - 第26條：每季結束提醒「應公開揭示」。
  - 第13條：決算金額 ≥ 3000 萬且有投資事業 → 提醒會計師簽證；第8條：收入 ≤ 300 萬 → 可僅置日記簿。
  - 第27條：每筆標保存到期年（憑證+5年、簿籍報告+10年）。
- 自動產生**法定報表（第6條）**：收支餘絀表、資產負債表（簡）、現金流量（簡）、淨值變動、財產清冊；**預算表（第12條）/決算表（第13條）**框架。
- 資料只存 localStorage；可**匯出/匯入 JSON＋CSV**（備份、交接、給會計師）。

### B. 公開透明版（會員查詢／政府查詢／經得起查）
- 唯讀的「按季公開」頁（第26條）：每季收支彙總、累計餘絀、準備金、重大支出明細、政府補助專區（第30條第8款）。
- 「查核就緒」清單（第27、28、30條）：保存年限狀態、可供查核之憑證/帳簿/報告/預決算/財產清冊對照。
- 幹部一鍵「匯出公開版」（去識別到適當程度、不含私人資料），自行決定發佈管道；**不自動 commit 真實數字**。

### 安全/隱私
- 沿用站內輕量密碼鎖模式（後台寫入需密碼；公開版唯讀）。真實數字不入 repo。

## 8. 雜湊存證 / 數位指紋（2026-06-17 續做）

- **原理**：每張單據照片以瀏覽器內建 `crypto.subtle.digest('SHA-256')` 算 64 字指紋（純本機、不上網、無相依）。照片改一像素指紋全變；可公開、不可倒推原圖。
- **receipt.html**：`sha256()`；存檔時 `rec.hash=await sha256(rec.img)`；檢視 modal 顯示指紋＋「🔒 驗證未竄改」鈕（`verifyRec` 重算比對 ✅/⛔）。
- **finance.html**：`sha256()`；明細列 📎單據旁加「🔒 驗證」(`verifyReceipt`)；`viewReceipt` 顯示指紋；`genReceiptPDF` 季度PDF加「數位指紋清單（SHA-256）」整段＋理事長/監事/日期簽章欄（§20·§28，使指紋存在時間有人證）。
- **誠實限制**（已向使用者說明）：雜湊只證「未竄改」，不單獨證「時間」（要可信時戳 TSA／網路，環境受限不做）；務實替代＝季度PDF指紋清單給理事長＋監事簽名用印。也防不了「一開始就拍假單據」（靠正本＋§27留存）。
- 已驗證：clean→✅、竄改→⛔ 偵測、PDF 含完整指紋，全 PASS。
