# 自動化測試

純前端網站的端對端測試（用 Playwright 真的開瀏覽器點點看），涵蓋四類：
- **A 功能**：加班費三種試算、薪資明細加總、入會 ROI
- **B 互動流程**：連署前彈窗、FAQ 小幫手、進度條、線上會議、章程快查卡
- **C 幹部後台**：名冊新增會員、案件台新增案件
- **D 流暢/導覽**：返回鍵、本機暫存、頁面載入速度
- **E 錯誤/邊界**：空白、除數 0、超大數字、非數字、空輸入不崩潰

## 怎麼跑

```bash
bash tests/run.sh
```

會自動開本機伺服器、跑完、收尾。全部通過離開碼 0；有失敗離開碼 1（改完程式想確認沒弄壞別頁，重跑這個就好）。

## 環境

預設用本沙箱的 Chromium 與 playwright-core 路徑；換環境可用環境變數覆寫：
`CHROME`、`PW_CORE`、`PORT`、`NODE`、`NODE_PATH`（見 `run.sh` 與 `e2e.cjs` 開頭）。

## 加新測試

打開 `e2e.cjs`，照現有 `await T('名稱', async p => { ... })` 格式加一條即可；
`ok(訊息)` 記成功、`bad(名稱, 原因)` 記失敗。
